1. Что такое PHP?

Из предисловия к руководству:

PHP - это скриптовый язык, который может быть встроен в HTML. Синтаксис языка в большинстве заимствован из C, Java и Perl c добавлением нескольких специфичных для PHP особенностей. Цель языка - позволить web-разработчикам легко создавать динамически генерируемые страницы.

Хорошее введение в PHP, написаное Stig Sæther Bakken, можно найти по адресу http://www.zend.com/zend/art/intro.php. Также большинство материалов PHP Конференций свободно доступно.

2. Что PHP означает?

PHP означает PHP: Hypertext Preprocessor. Это название смущает многих людей, так как первое слово в акрониме также является акронимом. Такой тип акронимов называется рекурсивным. Любопытствующие могут посетить Free On-Line Dictionary of Computing для дополнительной информации по рекурсивным акронимам.

3. Какая связь между версиями PHP?

PHP/FI 2.0 - ранняя и более неподдерживаемия версия PHP. PHP 3 - улучшенная замена PHP/FI 2.0. PHP 4 - текущая версия, которая использует Zend engine. PHP 5 использует Zend engine 2, которая, кроме прочего, предлагает множество дополнительных возможностей для объектно-ориентированного программирования .

4. Могу ли я одновременно запускать несколько версий PHP?

Да. Смотрите файл INSTALL из дистрибутива исходников PHP 4. Также читайте соответствующее приложение.

5. Какие отличия между PHP 3 и PHP 4?

На эту тему авторами PHP 4 написано несколько статей. Вот только некоторые из более важных новых возможностей:

  • Модуль расширенного API

  • Обобщённый процесс сборки под Unix

  • Общий интерфейс для web серверов, который также поддерживает многопоточные web сервера

  • Улучшенный выделитель синтаксиса

  • Поддержка родной HTTP сессии

  • Поддержка буферизованного вывода

  • Более мощная система конфигурации

  • Подсчёт ссылок

Пожалуйста, смотрите обзор Что нового в PHP 4 для детального объяснения этих возможностей и прочего. Если вы переходите из PHP 3 в PHP 4, также читайте соответствующее приложение.

6. Я думаю, Я нашёл ошибку! Кому я должен сказать?

Вам надо обратиться к базе данных ошибок в PHP и убедиться, что ошибка ещё не известна. Если вы не видите её в базе данных, заполните форму сообщения об ошибках. При занесении в базу данных, ошибка получает номер, по которому вы можете отслеживать её статус, поэтому важно использовать базу данных, а не просто послать сообщение в один из списков рассылки. База данных ошибок может быть найдена по адресу http://bugs.php.net/.
